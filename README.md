# Sgr A\* Simulation

Interactive WebGL visualization of a GRMHD accretion flow around
Sagittarius A\*. Real ray-traced radiation snapshots from a public
simulation library are mapped through three colormaps in a single
split-view canvas, with shader-drawn iso-contours, draggable panel
separators, and an orbit camera.

![Demo](public/grmhd/demo.gif)

## What's on screen

The plot is divided into three regions by two draggable separators:

| Region                                  | Quantity                  | Colormap                |
| --------------------------------------- | ------------------------- | ----------------------- |
| `x < uXSplit, y > uYSplit`              | Radiation Power           | viridis                 |
| `x < uXSplit, y < uYSplit`              | Magnetic Field Strength   | inferno                 |
| `x > uXSplit`                           | Density                   | black → orange → yellow |

Other layers:

- **Black-hole shadow** at the center, with the curved photon-ring lensing
  visible as a dark arc cutting through the orange plasma — this is GR
  light-bending baked into the data by the BHOSS ray-trace.
- **White iso-contours** on the left two panels. Drawn in-shader using
  `fwidth()` so the band thickness stays constant in screen pixels.
  Because the contours are derived from the same field that colors the
  background, they trace the actual bright filaments.
- **Three colorbars**, one in the middle of each region, showing the
  scale magnitudes.
- **Bottom scrubber** + play/pause for the 3-minute loop.

## Interaction

- **Camera** — drag to orbit, right-drag to pan, scroll to zoom. The
  rotate-counterclockwise button (top-right) resets to the canonical
  front-on view.
- **Separators** — two thin bars with diamond grips at the panel
  boundaries. Drag horizontally to slide the Density / Radiation+|B|
  split anywhere from −60 to +60 r_g; drag vertically to slide the
  Radiation / Magnetic Field Strength split (left panel only).
- **Crosshair button** (top-right) snaps both separators back to (0, 0).
- **Real GRMHD / Procedural** pill toggles between sampling the baked
  GRMHD atlas and a fully procedural noise + analytic-stream-function
  fallback.

## Running

```bash
npm install
npm run dev
# open http://localhost:3000 (or PORT=3010 npm run dev if 3000 is taken)
```

On first load a "tracing magnetic field lines…" bar shows for ~1 second
while a Web Worker precomputes the streamline geometry. After that the
simulation runs at vsync.

## Data

`public/grmhd/` ships three pre-baked artifacts loaded at runtime:

- `inu_atlas.png` (~1.4 MB) — 101-frame atlas of ray-traced radiation
  intensity at 256×128 per tile, sqrt-clip normalized so the
  frame-to-frame brightness variation of the MAD eruption cycle stays
  visible. Laid out in an 11×10 grid.
- `radial_profiles.json` (~1.9 MB) — 120-frame time series of
  radially-resolved density / B² / synchrotron emissivity profiles.
- `manifest.json` — atlas layout, normalization ranges, and plot domain.

Generated by `scripts/grmhd-to-textures.py` from two public CC-BY-4.0
datasets — please cite if you reuse:

- **Yoon, D., Chatterjee, K., Markoff, S., et al. (2020).** *Spectral
  and Imaging properties of Sgr A\* from High-Resolution 3D GRMHD
  Simulations with Radiative Cooling.* MNRAS 499, 3178.
  arXiv:2009.14227. Data: [Zenodo 3988208](https://zenodo.org/records/3988208).
- **Sosapanta-Salas, L. D., Liska, M., Markoff, S., et al. (2025).*
  Two-temperature treatments in magnetically arrested disk GRMHD
  simulations more accurately predict light curves of Sagittarius A\*.*
  MNRAS, [staf240](https://doi.org/10.1093/mnras/staf240).
  Data: [Zenodo 14793884](https://zenodo.org/records/14793884).

To regenerate the textures (after downloading the source NPZs from
the Zenodo links):

```bash
python3 -m venv venv && source venv/bin/activate
pip install numpy pillow
python scripts/grmhd-to-textures.py \
  --yoon /tmp/yoon --salas /tmp/salas --out public/grmhd
```

To regenerate `public/grmhd/demo.gif` (requires `ffmpeg` and the dev
server running on :3010):

```bash
node scripts/capture-gif.mjs --duration 9 --fps 10 --out public/grmhd/demo.gif
```

## Performance notes

- A Web Worker precomputes every streamline frame at startup; per-frame
  the main thread just swaps which `BufferGeometry` the `LineSegments`
  object renders.
- Simulation time lives in a `useRef` and updates the shader uniform
  via `useFrame`, so the React tree doesn't reconcile every animation
  frame.
- The Inu atlas is sampled with **linear interpolation between adjacent
  frames** so the plasma evolves with constant change rate rather than
  pulsing at frame boundaries.

## Tuning

The look lives in two places:

- `shaders/lib/fields.ts` — procedural-mode physics (density / B /
  radiation power and the analytic stream function).
- `shaders/simulation.frag.ts` — colormaps and blending for both modes,
  iso-contour spacing, per-quadrant gamma.
